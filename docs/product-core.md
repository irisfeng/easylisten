# 轻听产品核心：每天开始前重温

> 接送路上，给孩子听见更大的世界。

本文是轻听的产品北极星，也是每日精选、听稿编辑、音频生成、界面设计和功能迭代的最高层执行依据。每天的自动出刊必须读取本文；任何新功能、选题规则和体验优化，都先回答它是否更好地服务了下述用户与需求。

## 1. 轻听给谁听

### 主要收听者：6–16 岁的孩子和少年

- **6–9 岁**：从人物、故事和可观察的现象进入真实世界，需要具体、清楚、情绪安全的解释。
- **10–12 岁**：开始理解机制、历史背景和不同观点，需要建立知识之间的联系。
- **13–16 岁**：能够接触真实世界的复杂度，需要学习辨认证据、语境、论证和观点差异。

他们不是被动接受“儿童内容”的人。轻听尊重他们的理解力，不幼稚化，不用说教、作业和打卡换取注意力。

### 关键决策者：家长

家长负责第一次选择、信任判断和账号管理。家长真正需要的不是更多内容，而是：

- 不必每天亲自从新闻、杂志、播客和信息流中筛选；
- 能清楚知道内容来自哪里、为什么值得听、适合哪个年龄段；
- 放心让孩子独立收听，不担心事实失真、情绪刺激或隐性说教；
- 在接送、通勤、吃饭和周末出行等碎片时间里，低成本地打开就听。

因此，轻听同时服务两种成功：**孩子愿意听、能够听懂并产生问题；家长愿意信任并持续交给轻听筛选。**

## 2. 轻听解决什么问题

真实世界的优质内容很多，但它们分散、篇幅不一、面向成人阅读，也混在热点、广告和低质量信息里。孩子很难独立判断，家长也没有时间每天完成筛选、核验、解释和播放准备。

轻听接过这条费时的链路：

**从专业、可信的真实来源发现内容 → 严格筛选和核验 → 按年龄重组为适合耳朵理解的听稿 → 生成稳定、连续、可控制的音频 → 每天交付一份少而精的节目单。**

我们不是替孩子隔绝复杂世界，而是给他们一条安全、可信、逐步深入的入口。

## 3. 必须做好的核心功能

### 3.1 可信内容精选

- 先从专业、优质、可追溯的来源中寻找值得听的内容，再决定选题；不能先编一个热点或观点，再搜索材料填充。
- 新闻时事只是内容的一部分。科学机制、人文故事、社会观察、技术、生活知识、文化与体育精神共同构成更大的世界。
- 国内现实与国际视野动态均衡；热榜和 RSS 只负责发现，不能代替编辑判断。
- 少而精，宁缺毋滥，不为篇数、地域、双语或热点比例降低质量门槛。

### 3.2 少年适配的听稿编辑

- 忠于原始材料，事实、数字、因果和引语可以回到原文核验。
- 不是照读网页、机械摘要或伪装成原文翻译，而是为“听”重新组织顺序、背景和解释。
- 每篇逐一判断适龄段；既不把成人稿直接交给孩子，也不把少年内容写得幼稚。
- 孩子听完应该知道发生了什么、为什么值得关心，并愿意继续追问。

### 3.3 可靠的移动收听体验

- 手机网页、PWA 和微信内打开都应能迅速开始收听，未登录也能完整使用。
- 整篇连续播放、逐句高亮、点句即读、自动跟随、锁屏与后台控制是基础能力，不是装饰功能。
- 音色、语速和中英文切换必须清楚、稳定，不让用户面对技术性的声音列表或含混状态。
- 来源、适龄信息和播放控制在小屏上仍然清楚，单手可以操作。

### 3.4 每日有限节目单

- 每天交付少量真正值得听的内容，帮助家庭直接开始，而不是制造新的无限信息流。
- 当日刊优先，精选与存档承接长期价值；重复事件只有出现重要新进展时才再次进入。
- 主编的话、分享、收藏和个性化都服务于理解、选择和持续收听，不能反过来绑架内容判断。

## 4. 哪些不是核心

- **不是成人新闻 App**：不追求快讯覆盖、热搜搬运和资讯数量。
- **不是 AI 自动写作产品**：AI 可以辅助筛选、转述和核验，不能脱离可靠原文制造内容。
- **不是课程或学习打卡工具**：不靠任务、积分、排名和焦虑推动使用。
- **不是英语学习 App**：中英双语仍是最多两篇的实验能力，质量与收听价值优先。
- **不是声音展示工具**：男女声、语速和 TTS 技术只为让内容更自然、更易听。
- **不是算法无限流**：个性化只调整已通过编辑筛选的内容，不突破共同质量底线。

## 5. 每次更新前的五个问题

选题、写稿、设计、开发和运营在做决定前都必须回答：

1. 这会让哪个年龄段的孩子更愿意听、更容易听懂，还是只让功能看起来更多？
2. 这是否减少了家长筛选和判断的负担，并增强了信任？
3. 内容能否回到专业、可信、可追溯的原始来源？
4. 它是否改善了从打开、播放、理解到听完的核心收听链路？
5. 如果会损害准确性、适龄性、安静克制或使用可靠性，是否应该不做？

只有同时守住 **孩子的理解与好奇、家长的信任与省心、内容的真实与品质、收听的稳定与自然**，才是符合轻听方向的优化。
