# 信息源扩充评估（2026-07-19）

这次不按榜单热度批量导入 RSS，而把候选源依次经过四道门槛：源站身份与稳定性、
近期更新、可获得正文、脱离屏幕仍值得听。通过召回不等于入选，单篇文章仍须达到
80 分编辑门槛，并接受时效、重复事件和每日结构约束。

## 参考项目的定位

- `top-rss-list` 适合作为中文源发现目录，但收录混有失效链接、第三方 RSS 代理、
  社区热榜和商业资讯，排名不代表轻听的编辑质量。
- `TrendRadar` 主要提供热榜监测和趋势信号；默认热榜不是原始文章源，不直接进入
  轻听源库。它的分源时效、RSS 验活和热点与正文分离思路，现有管线已覆盖。
- `ai-daily-digest` 的英文个人博客池质量较高，但大量内容偏代码、工具发布或必须
  看图看表。只吸收其中适合转成听稿的原创分析和长访谈。

## `top-rss-list` 全量核查结论

已用 `npm run audit:sources` 对目录中的 340 条地址逐条核查，而不是抽样：目录没有
完全相同的 URL 重复；53 条通过 HTTPS、可解析和近 180 天更新三项技术检查，227 条
依赖 RSSHub、AnyFeeder、Feed43 等第三方代理，30 条仍是明文 HTTP，14 条失效，
13 条超过 180 天未更新，3 条已不再返回 RSS/Atom。

技术通过不等于适合孩子。对 53 条可用源再按发布者身份、原创性、正文可得性、
6–16 岁适配和纯听价值复核后：

- 现有源库已包含其中少数真正合适的来源，如少数派、爱范儿、游戏研究社、
  Solidot、阮一峰等；它们仍只是候选召回源，不保证单篇入选。
- 美团技术、V2EX 及多数开发者博客偏代码、架构图或程序员语境；虎嗅、36Kr、
  IT之家等商业与产品资讯噪声较高；热榜、社区和微博型 feed 缺少稳定的专业原文。
- 维基百科优良条目、豆瓣书评、Anyway.FM 等有发现价值，但发布责任、授权正文或
  事件时效不能满足正式源链路，暂不接入。

因此没有为了“榜单通过数”批量导入剩余来源。全量审计脚本保留在仓库，之后可以
重复运行并观察状态变化；正式源库继续执行更高一层人工准入。

## 明确不加入

- `top-rss-list` 中的 AnyFeeder/RSSHub 等第三方代理：存在 404、缺少发布时间或
  代理依赖，不能作为正式源链路。
- 美团技术团队：源可用，但文章高度依赖代码、架构图和产品发布，不适合纯听。
- 理想生活实验室：更新稳定，但近期品牌与消费资讯占比偏高。
- 卢昌海：当前 feed 主要是微博/Threads 更新，不是稳定长文流。
- Derek Thompson 的 Atlantic 作者 feed：可用但近期更新频率不足日刊召回。
- Pluralistic：文章有价值但立场型评论浓度高，而现有社会类源已经最拥挤；先留在
  观察名单，不为扩量而扩量。
- TrendRadar 的知乎、微博、抖音等热榜：只适合作为选题雷达，不是可追溯的专业
  原文源，不能替代来源核验。

## 后续淘汰规则

每次日刊记录分源抓取、入围和最终入选情况。连续 30 天无可用候选、正文长期抓取
失败，或持续产出不适合听读的内容时，降权或移除；反之，观察名单只有在稳定性与
实际文章质量均通过后才进入正式源库。
