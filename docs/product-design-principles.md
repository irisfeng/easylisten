# 轻听少年刊：产品与设计硬要求

## 1. 移动 Web 优先

- 以 360–430px 宽度和单手操作为第一验收环境，桌面端只做留白与宽度扩展。
- 首屏先回答“今天听什么、适合谁、为什么值得听”，主要播放操作固定在拇指热区。
- 点击目标不小于 44px；不依赖 hover；弱网、锁屏、切后台后能继续或恢复。
- 账号不是试听门槛。未登录可完整收听，登录只用于跨设备保存年龄段、收藏和进度。

## 2. 来源透明与准确性

- 来源是内容的一部分，不是页脚免责声明。标题之后、播放之前必须展示来源卡。
- 每篇正式内容必须包含：标准发布者名称、原文标题、可直达原文的链接、原文发布时间、核验时间和素材状态。
- 抓不到完整原文、链接失效、标题或发布者不清晰时，不生成正式听稿；宁可当天少一篇。
- 听稿是轻听编辑部的独立转述，不伪装成原文翻译。事实、数字、因果和引语必须能回到原文核验。
- 观点与事实分开表达；健康、未成年人、灾害等高风险内容增加二次来源核验。

## 3. 少年而不幼稚

- 年龄段只有 6–9、10–12、13–16；不收集孩子姓名、学校和生日。
- 不做课程、任务、积分与打卡。通过长期选材潜移默化培养理解力、好奇心、判断力和表达感。
- 国内现实、世界视野、科学机制、人文故事与体育精神共同组成每日内容，不把“少年内容”等同于校园资讯。
